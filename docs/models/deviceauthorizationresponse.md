If the value of<br/>the request parameter is valid, {@link #getGrantSubject()} will return<br/>the subject of the user who has given the grant to the client application.<br/>Authorization server implementations may use the value returned from<br/>{@link #getGrantSubject()} in order to determine the user to authenticate.<br/><br/>The user your system will authenticate during the authorization process<br/>(or has already authenticated) may be different from the user of the<br/>grant. The first implementer's draft of "Grant Management for OAuth 2.0"<br/>does not mention anything about the case, so the behavior in the case is<br/>left to implementations. Authlete will not perform the grant management<br/>action when the `subject` passed to Authlete does not match the<br/>user of the grant.<br/> |
